## Github Flow

Main is the source of truth and other branches are created from main. 
The branches are named according to the type of work being done, and are then PR'd back into main when the work is complete. 
The branches are deleted after the PR is merged.
These are all the possible branches that can be created from main:
- main
- feature/*
- fix/*
- refactor/*

At the end of every sprint we will push a tag and at the end of the project we will push a release.
